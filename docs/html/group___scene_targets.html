Detailed Description

Access scene elements with handy syntax.

These function are for accessing scene elements by using Scene Target syntax. Scene Target syntax is nicer than Scene DOM Path syntax.

Theses functions are part of the RenderToolbox3 batch renderer. Users should not use these functions directly. Just use the BatchRender() function.

Get renderer-native declaration and defaults, for a generic declaration.

Parameters
genericInfo
hints

Converts a generic object declaration info struct into to as struct array with renderer-native object declaration and default properties.

genericInfo must be an info struct as returned from ParseGenericMapping().

hints must be a struct of batch renderer hints, as returned from GetDefaultHints().

Returns a struct array of renderer-native mappings info, with the same fields as genericInfo. The first element of the returned struct will contain the renderer-native category name and type name that correspond to the given generic categorty and type.

Subsequent elements of the returned struct array will contain default, renderer-native property names, type names, and values.

The returned struct will contain a new 'genericName' field, which may contain a generic object type name or property name. This establishes the mapping between generic names and renderer-native names.

Used internally by BatchRender().

Usage: nativeInfo = GetNativeObjectDefaults(genericInfo, hints)

info ParseSceneTarget ( map  )

Scan a mappings struct for Scene Target info.

Parameters
map

Scan the given mapping struct map, for scene target syntax. map must be a mapping struct as returned from ParseMappings(). The mappings left hand value should contain Scene Target syntax.

Returns a struct of information about the Scene Target. This "info struct" is an intermediate form, used when converting mappings syntax and generic scene elements. The info struct will have the following fields:

  • id - unique identifier of a scene element
  • name - hint, category, or property of the scene element
  • type - the element type or property type
  • operator - operator from map
  • value - right-hand value from map
  • isDeclaration - whether map declares a new scene element See the Rendertoolbox3 wiki for more about Scene Targets .

Used internally by BatchRender().

Usage: info = ParseSceneTarget(map)

Convert mappings from Scene Target syntax to Scene DOM syntax.

Parameters
sceneTargetMaps
hints

Converts the given scene target mappings, which must contain scene target syntax on the left hand side, to renderer-native mappings that contain adjustment file scene DOM paths on the left hand side.

The given sceneTargetMaps must be a struct array of mappings as returned from ParseMappings(). It should have scene target syntax on the left hand side.

hints should be a struct of batch renderer hints, as returned from GetDefaultHints().

Converts scene target syntax to scene DOM paths that can be applied to renderer adjustments files, and returns one or more mappings structs that contains renderer-native scene paths on the left hand side.

If the block name in sceneTargetMaps.block matches the name of the current renderer in hints.renderer, then the names, types, and values given in the scene target syntax are preserverd in the renderer-native scene paths.

If the block name in sceneTargetMaps.block is "Generic", then the names and types and values are treated as Generic Scene Elements. These are converted and supplimented with other default mappings, in an attempt to make all renderers produce comparable outputs.

Returns a struct array of one or more mappings, with renderer-native scene DOM paths on the left-hand side. These can be used to modify renderer adjustments files.

See the Rendertoolbox3 wiki for more about Scene Targets .

Used internally by BatchRender().

Usage: nativeMaps = SceneTargetsToDOMPaths(sceneTargetMaps, hints)
